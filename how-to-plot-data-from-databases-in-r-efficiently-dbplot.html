Dealing with large datasets that live in a database, plotting in R can be challenging. How to do it efficiently?

Using data about world-wide chart success of songs and albums, we create an in-memory database to get going quickly without having to worry much about infrastructure. Efficiency in plotting from databases is largely determined by how much data you need to pull into R‚Äôs memory. Many plots only require aggregated data, so doing calculations inside the database pays off in terms of data transfer requirements.

We look at bar plots to compare different approaches: An inefficient way where ggplot2 does the aggregation, then manual calculation inside the database, as well as using a dedicated function from the dbplot package by Edgar Ruiz.

Further examples include a histogram, a scatterplot and a powerful alternative, a raster plot.
